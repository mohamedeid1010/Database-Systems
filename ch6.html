01

يعني إيه SQL أصلاً؟

Overview · History · RDBMS

💬 الفكرة في سطر

SQL = Structured Query Language — اللغة المعيارية (the standard) للتعامل مع قواعد البيانات العلائقية (RDBMS).

💡 تشبيه الـ SQL زي ما تطلب من جرسون: "هاتلي كل العملاء اللي في القاهرة مرتبين بالاسم". انت بتقول عايز إيه مش إزاي تجيبه — والـ DBMS هو اللي بيجيبه.
🔎 اعرف أكتر — RDBMS و تاريخ SQL
RDBMS: نظام بيدير الداتا كمجموعة جداول، والعلاقات بينها بتتمثّل بـ قيم مشتركة (مش links).

تاريخ سريع: 1970 Codd اخترع الفكرة · 1974-79 IBM عملت System R + Sequel · 1979 Oracle أول DB تجاري بـ SQL · 1986 معيار ANSI · تحديثات 1989/1992/1999/2003/2006/2008.

🎁 فوايد لغة معيارية موحّدة Benefits

  • 1 تقليل تكاليف التدريب · 2 إنتاجية أعلى · 3 portability للتطبيقات.
  • 4 عمر أطول للتطبيق · 5 تقليل الاعتماد على vendor واحد · 6 تواصل بين الأنظمة.
02

بيئة الـ SQL والأوامر

Catalog · Schema · DDL/DML/DCL · سؤال 3

🏛️ Catalog و Schema

  • Catalog مجموعة schemas بتوصف الداتابيز كلها.
  • Schema الهيكل اللي فيه وصف الـ objects اللي عملها اليوزر (base tables, views, constraints).

🧩 الـ 3 أنواع أوامر — مهم جداً! سؤال 3

DDL

تعريف الداتابيز: CREATE / ALTER / DROP + الـ constraints.

DML

الشغل على الداتا: SELECT / INSERT / UPDATE / DELETE.

DCL

التحكم والصلاحيات: GRANT / REVOKE + commit.

💡 تشبيه تخيّل مطعم: DDL = بناء المطبخ والطاولات، DML = طبخ وتقديم الأكل اليومي، DCL = مين مسموح يدخل المطبخ ومين لأ.
03

إنشاء الجداول — CREATE TABLE

DDL · Constraints · PK/FK

🏗️ خطوات إنشاء الجدول

  • 1 حدّد data types للأعمدة.
  • 2 حدّد الأعمدة اللي تقبل/ماتقبلش null.
  • 3 حدّد الأعمدة الـ unique (candidate keys).
  • 4 حدّد الـ PK–FK.
  • 5 حدّد القيم الافتراضية (default).
  • 6 حدّد الـ constraints (domain).
  • 7 اعمل الجدول والـ indexes.
-- الشكل العام CREATE TABLE table_name ( column1 datatype [NOT NULL] [DEFAULT value], ..., CONSTRAINT pk_name PRIMARY KEY (col), CONSTRAINT fk_name FOREIGN KEY (col) REFERENCES parent(col) );
💡 افتكر: الـ PK مينفعش يكون NULL أبداً. والـ composite PK = أكتر من عمود في الـ PRIMARY KEY.
✍️ اكتب الكود · +10 XP
اكتب أمر CREATE TABLE لجدول CUSTOMER_T فيه CustomerID (رقم، NOT NULL، primary key) و CustomerName (نص).
✅ الإجابة النموذجية:
CREATE TABLE CUSTOMER_T ( CustomerID NUMBER NOT NULL, CustomerName VARCHAR2(25), CONSTRAINT Customer_PK PRIMARY KEY (CustomerID) );
✍️ اكتب الكود · +10 XP
اكتب CREATE TABLE لجدول ORDER_T فيه OrderID (PK)، OrderDate، و CustomerID كـ foreign key يشاور على CUSTOMER_T.
✅ الإجابة النموذجية:
CREATE TABLE ORDER_T ( OrderID NUMBER NOT NULL, OrderDate DATE, CustomerID NUMBER, CONSTRAINT Order_PK PRIMARY KEY (OrderID), CONSTRAINT Order_FK FOREIGN KEY (CustomerID) REFERENCES CUSTOMER_T (CustomerID) );
الـ FK بيطبّق الـ referential integrity: أي CustomerID في ORDER_T لازم يكون موجود في CUSTOMER_T.
04

تعديل وحذف الجداول

ALTER TABLE · DROP TABLE

🔧 ALTER و DROP

ALTER TABLE بيغيّر مواصفات الأعمدة (يضيف/يعدّل/يحذف عمود). DROP TABLE بيحذف الجدول كله من الـ schema.

ALTER TABLE CUSTOMER_T ADD CustomerType VARCHAR2(2) DEFAULT 'Commercial'; DROP TABLE CUSTOMER_T;
✍️ اكتب الكود · +10 XP
ضيف عمود جديد اسمه CustomerEmail نوعه VARCHAR2(50) لجدول CUSTOMER_T.
✅ الإجابة النموذجية:
ALTER TABLE CUSTOMER_T ADD CustomerEmail VARCHAR2(50);
✍️ اكتب الكود · +10 XP
احذف جدول ORDERLINE_T بالكامل من الـ schema.
✅ الإجابة النموذجية:
DROP TABLE ORDERLINE_T;
05

إدخال وتعديل وحذف الداتا

INSERT · UPDATE · DELETE (DML)

📥 الـ 3 أوامر الأساسية

-- INSERT: إضافة صف INSERT INTO CUSTOMER_T VALUES (001, 'Contemporary Casuals', 'Gainesville', 'FL'); -- لو في أعمدة null لازم تحدد الأعمدة: INSERT INTO CUSTOMER_T (CustomerID, CustomerName) VALUES (001, 'Casuals'); -- UPDATE: تعديل UPDATE PRODUCT_T SET ProductStandardPrice = 775 WHERE ProductID = 7; -- DELETE: حذف DELETE FROM CUSTOMER_T WHERE CustomerState = 'HI'; DELETE FROM CUSTOMER_T; -- يمسح كل الصفوف!
⚠️ خطر: DELETE FROM CUSTOMER_T; من غير WHERE = بيمسح كل الصفوف. خلي بالك من الـ WHERE!
✍️ اكتب الكود · +10 XP
عدّل اسم المنتج اللي ProductID = 4 وخلّيه 'Entertainment Center'.
✅ الإجابة النموذجية:
UPDATE PRODUCT_T SET ProductDescription = 'Entertainment Center' WHERE ProductID = 4;
✍️ اكتب الكود · +10 XP
احذف العميل اللي CustomerID = 16 من CUSTOMER_T.
✅ الإجابة النموذجية:
DELETE FROM CUSTOMER_T WHERE CustomerID = 16;
🔎 اعرف أكتر — Identity Column (سؤال 9)
الـ identity column بيعمل sequence رقمي تلقائي. مثال: ID SMALLINT IDENTITY(5,2) → أول صف بياخد 5، واللي بعده 7، 9... الفايدة: توليد رقم فريد أوتوماتيك (زي ID). ملحوظة: ده أمر SQL Server ومش بيشتغل في Oracle.
06

الاستعلامات — SELECT

SELECT · FROM · WHERE · COUNT · Alias

🔍 تركيب الـ SELECT

ده أهم أمر في الشابتر — ترتيب الجمل ثابت:

  • SELECT الأعمدة اللي عايز ترجّعها.
  • FROM الجدول/الجداول.
  • WHERE شروط اختيار الصفوف.
  • GROUP BY تجميع النتايج.
  • HAVING شرط على المجموعات.
  • ORDER BY ترتيب النتيجة.
-- مثال: المنتجات اللي سعرها أقل من 275 SELECT ProductDescription, ProductStandardPrice FROM PRODUCT_T WHERE ProductStandardPrice < 275;
💡 افتكر Comparison operators: = , <> , < , > , <= , >=. و Alias = اسم بديل للعمود/الجدول: SELECT CustomerName AS Name FROM CUSTOMER_T CUST;
✍️ اكتب الكود · +10 XP
هات ProductDescription و ProductFinish لكل المنتجات اللي سعرها أكبر من 300.
✅ الإجابة النموذجية:
SELECT ProductDescription, ProductFinish FROM PRODUCT_T WHERE ProductStandardPrice > 300;

🧮 Aggregate Functions — COUNT

دوال بتحسب على مجموعة صفوف: COUNT, SUM, AVG, MIN, MAX.

SELECT COUNT(*) FROM ORDERLINE_T WHERE OrderID = 1004;
💡 مهم (سؤال 4): COUNT بيعدّ الصفوف اللي فيها قيمة (بيتجاهل null) · COUNT DISTINCT بيعدّ القيم المختلفة بس · COUNT(*) بيعدّ كل الصفوف حتى لو فيها null. ومينفعش تحط عمود عادي مع aggregate إلا لو موجود في الـ GROUP BY.
✍️ اكتب الكود · +10 XP
اعدّ كام منتج موجود في جدول PRODUCT_T.
✅ الإجابة النموذجية:
SELECT COUNT(*) FROM PRODUCT_T;
07

الشروط — Boolean & LIKE

AND · OR · NOT · LIKE · IN · سؤال 5 و 7

🔀 AND / OR / NOT

ترتيب التنفيذ الافتراضي (سؤال 5): NOT الأول، بعدين AND، بعدين OR. عايز تغيّر الترتيب؟ استخدم الأقواس ().

SELECT ProductDescription FROM PRODUCT_T WHERE (ProductFinish = 'Cherry' OR ProductFinish = 'Walnut') AND ProductStandardPrice < 300;
💡 افتكر LIKE بيقارن النصوص بـ wildcards. '%Desk' = أي نص بينتهي بـ Desk. و IN (سؤال 7) بيستخدم في الـ WHERE وبعده قايمة قيم بين أقواس، وممكن تستبدله بـ OR.
✍️ اكتب الكود · +10 XP
هات ProductID و ProductDescription لكل المنتجات اللي وصفها بينتهي بكلمة Desk.
✅ الإجابة النموذجية:
SELECT ProductID, ProductDescription FROM PRODUCT_T WHERE ProductDescription LIKE '%Desk';
✍️ اكتب الكود · +10 XP
هات كل العملاء اللي الـ CustomerState بتاعهم واحدة من: FL, TX, CA, HI (استخدم IN).
✅ الإجابة النموذجية:
SELECT * FROM CUSTOMER_T WHERE CustomerState IN ('FL', 'TX', 'CA', 'HI');
نفسها بالـ OR: WHERE CustomerState = 'FL' OR CustomerState = 'TX' ...
08

التجميع والترتيب

ORDER BY · GROUP BY · HAVING · سؤال 6 و 8 و 10

↕️ ORDER BY · GROUP BY · HAVING

  • ORDER BY بترتّب النتيجة (سؤال 8). تشاور للعمود بـ اسمه أو بـ رقم ترتيبه في الـ SELECT (سؤال 10). و ASC/DESC.
  • GROUP BY بتجمّع الصفوف في فئات عشان الـ aggregate.
  • HAVING زي WHERE بس بتشتغل على المجموعات مش الصفوف (سؤال 6).
-- كل state وعدد عملائها، بس اللي عندها أكتر من عميل، مرتبة SELECT CustomerState, COUNT(*) FROM CUSTOMER_T GROUP BY CustomerState HAVING COUNT(*) > 1 ORDER BY CustomerState;
💡 تشبيه HAVING الـ WHERE بيفلتر الأفراد قبل التجميع، والـ HAVING بيفلتر المجموعات بعد التجميع. زي: "هات الفصول اللي متوسطها أعلى من 80" — ده شرط على المجموعة.
✍️ اكتب الكود · +10 XP
هات أسماء العملاء (CustomerName) مرتبة أبجدياً تصاعدي.
✅ الإجابة النموذجية:
SELECT CustomerName FROM CUSTOMER_T ORDER BY CustomerName;
✍️ اكتب الكود · +10 XP
هات الـ CustomerState وعدد العملاء في كل state، بس للـ states اللي فيها أكتر من 2 عملاء.
✅ الإجابة النموذجية:
SELECT CustomerState, COUNT(*) FROM CUSTOMER_T GROUP BY CustomerState HAVING COUNT(*) > 2;
09

الـ Views والـ Indexes

CREATE VIEW · CREATE INDEX · CHECK

🪟 Views

View = جدول افتراضي (virtual) بيتعمل من query على جداول تانية. مفيش داتا متخزّنة فعلاً — بتتجمّع وقت الطلب (dynamic view).

CREATE VIEW Expensive_V AS SELECT ProductID, ProductDescription, ProductStandardPrice FROM PRODUCT_T WHERE ProductStandardPrice > 300;

✅ مزايا

تبسّط الـ queries · تساعد في الأمان · إنتاجية · داتا محدّثة · مساحة قليلة · استقلالية فيزيائية.

VS

⚠️ عيوب

بتاخد وقت معالجة كل مرة تُستخدم · مش دايماً قابلة للتحديث (updateable).

🔎 اعرف أكتر — CHECK و WITH CHECK OPTION (سؤال 11)
في CREATE TABLE: الـ CHECK بيحط validation rule للقيم المسموحة (مثلاً ProductFinish لازم تكون من قايمة معيّنة). في CREATE VIEW: الـ WITH CHECK OPTION بيرفض أي INSERT/UPDATE من خلال الـ view لو بيخالف شرط الـ view.

⚡ CREATE INDEX

الـ index بيسرّع الوصول للداتا (زي ما اتعلمنا في شابتر 5).

CREATE INDEX NAME_IDX ON CUSTOMER_T (CustomerName);
✍️ اكتب الكود · +10 XP
اعمل view اسمه CustomerNames_V بيرجّع CustomerID و CustomerName من CUSTOMER_T.
✅ الإجابة النموذجية:
CREATE VIEW CustomerNames_V AS SELECT CustomerID, CustomerName FROM CUSTOMER_T;
✍️ اكتب الكود · +10 XP
اعمل index اسمه PRICE_IDX على عمود ProductStandardPrice في PRODUCT_T.
✅ الإجابة النموذجية:
CREATE INDEX PRICE_IDX ON PRODUCT_T (ProductStandardPrice);

فلاش كاردز — التعريفات

دوس الكارت يتقلب · سؤال 1

التعريفات بنص الشيت بالحرف — دوس الكارت يوريك الإجابة 👇

Base table
دوس ↻
A table in the relational data model containing the inserted raw data; likely corresponds to one physical file in secondary storage.
Data Definition Language
دوس ↻
Commands used to define a database: creating, altering, and dropping tables and establishing constraints.
Data Manipulation Language
دوس ↻
Commands used to maintain and query a database: updating, inserting, modifying, and querying data.
Dynamic view
دوس ↻
A virtual table created dynamically upon request; its definition is stored in the system catalog and its contents are materialized as the result of an SQL query.
Referential integrity constraint
دوس ↻
A constraint that the value/existence of an attribute depends on a primary key value; a many-side value must match a one-side PK value, or be null.
Relational DBMS (RDBMS)
دوس ↻
A DBMS that manages data as a collection of tables in which all relationships are represented by common values (not links) in related tables.
Schema
دوس ↻
A structure that contains descriptions of objects created by a user — base tables, views, constraints — as part of a database.
Virtual table
دوس ↻
A table constructed automatically as needed by a DBMS; not maintained as real data. Sometimes referred to as a dynamic view.

12 Database descriptions — CREATE TABLE (P&E) ›
Problems & Exercises (book p.283–284): Write the database description for the given relations using CREATE TABLE (Student, Faculty, Course, Section, Qualified, Registration).
Note: A particular SQL system may restrict the length of column names or may not permit embedded spaces in names. The following answer may not be acceptable to the SQL system you use; please modify the grammar accordingly.
CREATE TABLE Student (StudentID NUMBER NOT NULL, StudentName VARCHAR2(25), CONSTRAINT Student_PK PRIMARY KEY (StudentID)); CREATE TABLE Faculty (FacultyID NUMBER NOT NULL, FacultyName VARCHAR2(25), CONSTRAINT Faculty_PK PRIMARY KEY (FacultyID)); CREATE TABLE Course (CourseID CHAR(8) NOT NULL, CourseName VARCHAR2(15), CONSTRAINT Course_PK PRIMARY KEY (CourseID)); CREATE TABLE Section (SectionNo NUMBER NOT NULL, Semester CHAR(7) NOT NULL, CourseID CHAR(8), CONSTRAINT Section_PK PRIMARY KEY(SectionNo), CONSTRAINT Section_FK FOREIGN KEY (CourseID) REFERENCES Course (CourseID)); CREATE TABLE Qualified (FacultyID NUMBER NOT NULL, CourseID CHAR(8) NOT NULL, DateQualified DATE, CONSTRAINT IsQualified_PK PRIMARY KEY (FacultyID, CourseID), CONSTRAINT QualifiedFaculty_FK FOREIGN KEY (FacultyID) REFERENCES Faculty (FacultyID), CONSTRAINT QualifiedCourse_FK FOREIGN KEY (CourseID) REFERENCES Course (CourseID)); CREATE TABLE Registration (StudentID NUMBER NOT NULL, SectionNo NUMBER NOT NULL, CONSTRAINT IsRegistered_PK PRIMARY KEY (StudentID, SectionNo), CONSTRAINT StudentIsRegistered_FK FOREIGN KEY(StudentID) REFERENCES Student(StudentID), CONSTRAINT CourseIsRegistered_FK FOREIGN KEY (SectionNo) REFERENCES Section(SectionNo));
13 Creating a view ›
Create a view that lists StudentID and StudentName.
CREATE VIEW Student_V AS SELECT StudentID, StudentName FROM Student_T;

Show the SQL clause used to establish referential integrity in DDL.
CONSTRAINT Section_FK FOREIGN KEY (CourseID) REFERENCES Course (CourseID));
15 SQL Definition commands (ALTER / DROP / MODIFY) ›
a) Add a Class column to Student_T. b) Drop the Registration_T table. c) Change FacultyName length to 40.

b.
DROP TABLE Registration_T;

17 SQL queries (SELECT / WHERE / MIN) ›
a) Students with StudentID < 50000. b) FacultyName where FacultyID = 4756. c) MIN SectionID where Semester = 'I-2016'.
a.
SELECT StudentID, StudentName FROM Student_T WHERE StudentID < 50000;
b.
SELECT FacultyName FROM Faculty_T WHERE FacultyID = 4756;
c.

18 SQL queries (COUNT / dates) ›
a) Count registrations for Section 2714, Semester 'I-2015'. b) Qualified rows with DateQualified ≥ 01-JAN-2008.
a.
SELECT COUNT(*) FROM Registration_T WHERE Section = 2714 AND Semester = 'I-2015';
b.
SELECT FacultyID, CourseID, DateQualified FROM Qualified_T WHERE DateQualified >= '01-JAN-2008';
19 SQL queries (GROUP BY / HAVING / subquery) ›
a) Students registered in both the Database (Section 2714) and Networking (Section 2715) courses. b) Instructors who can teach one but not both of ISM 3113 / ISM 3112. c) Courses taught in first semester but not the second.
a.
SELECT StudentID, COUNT(*) FROM Registration_T WHERE SectionNo IN (2714, 2715) GROUP BY StudentID HAVING COUNT(*) > 1;
b.
SELECT FacultyID, COUNT(*) FROM Qualified_T WHERE CourseID IN ('ISM 3113', 'ISM 3112') GROUP BY InstructorID HAVING COUNT(*) = 1;
c. (uses a Chapter 7 tool — subquery)
SELECT CourseID, Semester FROM Section_T WHERE Semester = 'I-2015' AND CourseID NOT IN (SELECT CourseID FROM SECTION WHERE Semester = 'II-2015');
20 SQL queries (DISTINCT / ORDER BY) ›
a) Distinct CourseIDs. b) Student names sorted. c) Registrations of 'I-2015' sorted. d) Courses sorted ascending.
a.
SELECT DISTINCT CourseID FROM Section_T;
b.
SELECT StudentName FROM Student_T ORDER BY StudentName;
c.
SELECT SectionNo, Semester, StudentID FROM Registration_T WHERE Semester = 'I-2015' ORDER BY SectionNo, Semester, StudentID;
d.
SELECT CourseID, CourseName FROM Course_T ORDER BY CourseID ASC;
21 SQL queries (WHERE / LIKE / HAVING / subquery) ›
a) SectionNo for ISM 3113 in 'I-2015'. b) Courses whose name starts with 'Data'. c) Faculty qualified for both ISM 3112 & ISM 3113. d) Same as c but DateQualified after 31-DEC-2005. (last) Faculty assigned to ISM 4212 sections in 'II-2015'.
a.
SELECT SectionNo FROM Section_T WHERE CourseID = 'ISM 3113' AND Semester = 'I-2015';
b.
SELECT CourseID, CourseName FROM Course_T WHERE CourseName LIKE 'Data%';
c.
SELECT FacultyID FROM Qualified_T WHERE CourseID IN ('ISM 3112', 'ISM 3113') GROUP BY FacultyID HAVING COUNT(CourseID) > 1;
d.
SELECT FacultyID FROM Qualified_T WHERE CourseID IN ('ISM 3112', 'ISM 3113') AND DateQualified > '31-DEC-2005' GROUP BY FacultyID HAVING COUNT(CourseID) > 1;
(last) — uses the Assignment table & a subquery
SELECT FacultyID FROM Assignment_T WHERE Semester = 'II-2015' AND SectionNo IN (SELECT SectionNo FROM Section_T WHERE CourseID = 'ISM 4212');

📖 دليل SQL السريع (Intro SQL)

🏗️ إنشاء وتعديل الجداول (DDL)

CREATE TABLE Student (ID INT PRIMARY KEY, Name STRING); ALTER TABLE Student ADD Class STRING; DROP TABLE Student;

📥 تعديل البيانات (DML)

INSERT INTO Student VALUES (1, 'Lopez', 'SO'); UPDATE Course SET CourseName = 'DB' WHERE CourseID = 'ISM 4212'; DELETE FROM Student WHERE ID = 1;

🔍 استعلام SELECT الأساسي

SELECT Name, Class FROM Student WHERE ID IN (1, 2) ORDER BY Name ASC;

🧮 التجميع والتصفية (Group By & Having)

SELECT State, COUNT(*) FROM Cust GROUP BY State HAVING COUNT(*) > 2;

🪟 الـ Views والـ Indexes

CREATE VIEW Student_V AS SELECT ID, Name FROM Student; CREATE INDEX Name_IDX ON Student (Name);
